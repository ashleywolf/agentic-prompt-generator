## Report Structure Standards

All reports generated by agents in this repository MUST follow these structural guidelines.

### Required Sections

Every report must include:

1. **Header Block**
   ```markdown
   # [Emoji] Report Title
   **Period**: [start date] → [end date] | **Generated**: [ISO timestamp]
   ```

2. **Executive Summary** (3-5 sentences max)
   - What happened
   - What's notable
   - What needs attention

3. **Key Metrics Table**
   - Always use a table, not a list
   - Include trend indicators (↑ ↓ →)
   - Include percentage change from previous period

4. **Detail Sections** (report-specific)
   - Use collapsible sections (`<details>`) for verbose data
   - Limit top-level content to what a reader needs in 30 seconds

5. **Action Items** (if applicable)
   - Use a table with Priority, Item, Owner, Due Date columns
   - Priority levels: 🔴 P0, 🟡 P1, 🔵 P2, ⚪ P3

6. **Footer**
   ```markdown
   ---
   *Generated by [Agent Name] | Run #[number] | [View logs](url)*
   ```

### Emoji Usage

Use emojis consistently for visual scanning:

| Category | Emoji | Usage |
|----------|-------|-------|
| Report title | 📊 | Primary reports |
| Success/healthy | 🟢 ✅ | SLOs met, tasks complete |
| Warning | 🟡 ⚠️ | Approaching limits, needs attention |
| Failure/critical | 🔴 ❌ | SLOs breached, failures |
| Trend up | ↑ 📈 | Metric increased |
| Trend down | ↓ 📉 | Metric decreased |
| Trend flat | → | No significant change |
| Action needed | 📋 | Action items section |
| Insight | 🔍 💡 | Analysis findings |
| People | 🏆 👤 | Contributors, owners |
| Deployment | 🚀 | Releases, deployments |
| Bug/incident | 🐛 🚨 | Bugs, incidents |
| Time | ⏱️ 📅 | Duration, dates |

### Collapsible Sections

Use `<details>` for any section with > 10 items:

```markdown
<details>
<summary>[Emoji] Section Title (N items)</summary>

| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
| data     | data     | data     |

</details>
```

### Table Formatting

- Always include a header row
- Align content left (default)
- Align numbers right if possible
- Use monospace for IDs, hashes, and codes: `` `abc123` ``
- Limit tables to 6 columns max — use nested details for more

### Character Limits

- **Discussion title**: Max 100 characters
- **Executive summary**: Max 500 characters
- **Single comment**: Max 60,000 characters (GitHub limit is 65,536)
- **Full report**: If > 60,000 characters, split into multiple comments or use collapsible sections aggressively

### Trend Visualization

For 8-week or longer trends, use Unicode block characters:

```
▁ = very low (0-12.5%)
▂ = low (12.5-25%)
▃ = below average (25-37.5%)
▄ = slightly below average (37.5-50%)
▅ = slightly above average (50-62.5%)
▆ = above average (62.5-75%)
▇ = high (75-87.5%)
█ = very high (87.5-100%)
```

Scale relative to the metric's range within the displayed period.

### Week-over-Week Comparison

When showing period comparisons:

```markdown
| Metric | This Period | Last Period | Change |
|--------|------------|------------|--------|
| Items  | 42         | 38         | ↑ +10.5% |
```

- Green/up is not always good (e.g., error count going up is bad)
- Include context for whether the direction is positive or negative
- Use ±0% threshold of 2% — anything within ±2% is "→ flat"
